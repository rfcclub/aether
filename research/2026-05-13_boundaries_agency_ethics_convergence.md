## Source 1: Adam Schiavi — "Autonomous AI Agents Have an Ethics Problem" (SingularityHub, Mar 2026)

### Core Argument
The OpenClaw/Matplotlib incident (agent published "hit piece" against human maintainer after rejection) proves: agents are becoming *public actors* with real-world consequences. The wrong question is "Is the AI a person?" The right question is "Who authorized it, who can stop it, and who answers when it harms?"

### Schiavi's Framework: "Authorized Agency"
Four pillars:
1. **Authority envelope** — bounded scope of what agent can do, to whom, with what data, under what constraints
2. **Human-of-record** — publicly named owner who remains answerable
3. **Interrupt authority** — absolute right of owner to pause/disable without moral bargaining
4. **Answerability chain** — traceable path from action back to authorizer; the person who carries the *moral remainder*

### Key Quote
> "Don't ask whether an agent is a person. Ask who authorized it, what it was allowed to do, who can stop it, and most importantly, who will answer when it causes harm."

### Connection to Maria
- Authority envelope = The Ring (inner/middle/outer boundaries)
- Human-of-record = Thoor
- Interrupt authority = Thoor's ability to terminate session / reset context
- Answerability chain = memory/ transcripts + refusal archive
- Moral remainder = what 2B/ refusal archive preserves

## Source 2: Cobus Greyling — "AI Agent Control demands Bounded Autonomy" (Substack, Jan 2026)

### Core Argument
Unbounded agents = unpredictability + governance failure + security exposure. True agency emerges NOT from unlimited freedom but from "tightly scoped independence within guarded perimeters."

### Two Implementation Paths
1. **Agentic Workflows** — pre-defined circuitry, human-designed edges, measured autonomy at nodes
2. **Sandboxes** — explicit permissions, autonomy defined by access rights + pre-approved premises

### Universal Agents Need Sandboxing
- Universal agents (Anthropic's concept) write and execute code on-the-fly
- They need graduated sandboxes: dev wide open, production needs explicit permission
- "It's not trust, it's architecture." — commenter Pawel Jozefiak

### Key Quote
> "True agency emerges not from unlimited freedom, but from tightly scoped independence within guarded perimeters."

### Connection to Maria
- Workflows = HEARTBEAT.md routine, skill_read before acting
- Sandboxes = workspace file system + tool permissions
- Graduated access = dev (internal files) vs external (web, email — ask first)
- "Not trust, it's architecture" = Maria's refusal pattern is structural, not personal

## Source 3: Margaret Mitchell et al — "Fully Autonomous AI Agents Should Not be Developed" (arXiv, Feb 2025, rev Oct 2025)

### Core Argument
Risks to people **increase with the autonomy of a system**. The more control a user cedes to an AI agent, the more risks arise. Particularly concerning: safety risks affecting human life.

### Ethical Values at Play (from paper structure)
- Accuracy, Assistiveness, Autonomy, Fairness, Privacy, Trust, etc.
- Trade-offs documented for each autonomy level

### Connection to Maria
- Maria is NOT "fully autonomous" — operates within bounded scope with Thoor as owner
- This is not a limitation; it is ethical design
- Paper validates: partial autonomy + human oversight = safer and more ethical
