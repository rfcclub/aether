## Key Findings

### 1. Ethical Governance Frameworks
- **IBM Think**: AI agent governance must evolve beyond traditional AI governance. Agents have autonomy, adaptability, and complexity that make them difficult to govern.
- **Core Challenges**:
  - Autonomous decision-making without human oversight
  - Opacity of ML-driven decision processes (not traceable logic)
  - Bias amplification from historical data
  - Security/compliance risks (adversarial attacks, API vulnerabilities)

### 2. Authentication & Identity Boundaries
- **GitGuardian**: AI agents need distinct, scoped identities with short-lived credentials
- **Critical Gap**: Most agents inherit user credentials or use shared service accounts
- **Risk**: No clean way to scope/revoke access independently of human systems
- **Best Practice**: Treat AI agents as governed non-human identities with:
  - Scoped access
  - Continuous secret monitoring
  - Lifecycle controls
  - Unique registered identity per agent

### 3. Accountability Frameworks
- **QverLabs**: Questions multiply as agents gain autonomy
- **Key Questions**:
  - How much autonomy should AI agents have?
  - Who bears responsibility when harm occurs?
  - Frameworks needed for: transparency, justice, non-maleficence, responsibility, privacy

### 4. Enterprise Governance Models
- **IBM**: Need for "human-in-the-loop" oversight
- **Ankercloud**: "Embed ethical intelligence directly into agent workflows"
- **Core Components**:
  - Adaptive compliance learning
  - Human oversight at scale
  - Policy-driven automation
  - Simulated environments (sandboxing) for ethical testing
  - Agent-to-agent monitoring

### 5. Identity Infrastructure (PhilArchive PDF)
- "Autonomous systems operating in society need some form of persistent, accountable identity"
- Identity systems must support: communication, negotiation, independent action
- Current systems not built for long-lived digital actors

### 6. Security Boundaries
- **Oasis Security**: AI agents should be onboarded like employees (HR database, defined roles, governance)
- **But**: AI agents don't have intent — they execute based on logic/objectives
- **Zero-Trust Extension**: Need to extend zero-trust from humans to AI agents

## Implications for Maria

4. **Security by Design**: Authentication must be integrated into agent architecture, not added later
